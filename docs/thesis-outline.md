# 毕业论文大纲

推荐题目：

**《面向 OI 竞赛的在线评测系统安全考试客户端设计与实现》**

## 第 1 章 绪论

- 在线评测系统在程序设计教学、竞赛训练和考试中的应用背景。
- OI 赛制对本地编辑、样例调试、远程隐藏数据判题的需求。
- 普通浏览器考试环境在切屏、复制粘贴、换机、换 IP 等方面的不足。
- 本课题的研究内容、工程目标和实际意义。

## 第 2 章 相关技术

- QingdaoU OnlineJudge：Django 后端、Vue 前端、JudgeServer/Judger、Docker Compose 部署。
- 桌面客户端技术：Electron/Chromium、Windows 全屏和置顶窗口、快捷键拦截。
- 本地 C++ 编译运行：MinGW/g++、临时工作目录、超时控制、标准输入输出。
- 监考审计：考试会话、心跳、违规事件、教师解锁。

## 第 3 章 需求分析

- 学生端：登录 OJ、进入比赛、查看题目、编写代码、本地运行样例、远程提交。
- 教师端：创建比赛、管理题目、设置 IP 限制、查看提交、处理违规锁定。
- 监考需求：禁止切屏、禁止复制粘贴、检测黑名单进程、锁定考试窗口。
- 安全边界：隐藏测试数据只保存在服务端，不在客户端落地。

## 第 4 章 系统设计

- 总体架构：自部署 OJ 服务端 + Windows 考试客户端 + 本地编译器。
- 客户端主界面：直接加载自部署 OnlineJudge 页面，保证和原 OJ 使用体验一致。
- 本地编辑器：随客户端启动自动打开，支持 C++14 编译和样例输入输出。
- 监考模块：全屏置顶、复制粘贴拦截、剪贴板清空、失焦锁屏、教师密码解锁。
- 服务端扩展：`ExamPolicy`、`ExamSession`、`ProctorEvent` 等审计模型。

## 第 5 章 系统实现

- OnlineJudge 从零部署与超级管理员初始化。
- Electron 客户端加载自部署 OJ 的实现。
- 本地样例运行器的实现。
- 监考策略的实现。
- 后端监考 API 原型实现。

## 第 6 章 测试与分析

- 部署测试：Docker Compose 启动、超级管理员创建、中文界面、访问 OJ。
- 客户端测试：默认打开 OJ、默认打开编辑器、禁止复制粘贴、切屏锁定。
- 本地编译测试：编译成功、编译错误、运行超时、运行错误、样例输出。
- 远程判题测试：ACM/OI 提交、排名更新、隐藏数据判题。
- 局限性分析：非内核级对抗、管理员权限依赖、进程检测绕过风险。

## 第 7 章 总结与展望

- 总结系统实现效果。
- 展望客户端签名、自动更新、服务端看板、摄像头辅助监考、更多语言支持。
